The `cypress-io/github-action` integrates Cypress testing into your GitHub Actions CI workflows with a single step. It handles dependency installation, caching, server startup, and test execution — so you can focus on writing tests.

## What the action does

In addition to running Cypress tests, the action handles the full CI setup lifecycle:

<CardGroup cols={2}>
  <Card title="Dependency installation" icon="download">
    Detects your lock file and installs dependencies with npm, pnpm, or Yarn automatically
  </Card>
  <Card title="Intelligent caching" icon="database">
    Caches the Cypress binary and package manager cache to speed up subsequent runs
  </Card>
  <Card title="Server coordination" icon="server">
    Start your dev server in the background and wait for it to respond before running tests
  </Card>
  <Card title="Parallel execution" icon="split">
    Load-balance tests across multiple containers using Cypress Cloud Smart Orchestration
  </Card>
  <Card title="Job summaries" icon="chart-bar">
    Automatically publishes pass/fail counts and duration to the GitHub Actions job summary
  </Card>
  <Card title="Docker support" icon="docker">
    Run tests inside Docker containers for consistent, reproducible CI environments
  </Card>
</CardGroup>

## Get started in three steps

<Steps>
  <Step title="Add Cypress to your project">
    Install Cypress as a dev dependency and make sure a lock file (`package-lock.json`, `pnpm-lock.yaml`, or `yarn.lock`) is committed to your repository.

    ```bash
    npm install --save-dev cypress
    ```
  </Step>
  <Step title="Add the action to your workflow">
    Create or update your GitHub Actions workflow file to include the Cypress action:

    ```yaml .github/workflows/ci.yml
    name: CI
    on: push
    jobs:
      cypress-run:
        runs-on: ubuntu-24.04
        steps:
          - uses: actions/checkout@v6
          - uses: cypress-io/github-action@v7
    ```
  </Step>
  <Step title="Push and watch your tests run">
    Push your changes to GitHub. The action will install dependencies, cache them for future runs, and execute your Cypress tests automatically.
  </Step>
</Steps>

<Tip>
  Use `cypress-io/github-action@v7` to always run the latest patch release on the v7 branch. Pin to a specific tag like `@v7.0.0` if you need tighter version control.
</Tip>
