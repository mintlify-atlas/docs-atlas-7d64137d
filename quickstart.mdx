This page walks you through adding `cypress-io/github-action` to a repository that already has Cypress installed. The action installs your dependencies, caches them, and runs your Cypress tests in a single step.

<Note>
  The action requires a lock file (`package-lock.json`, `pnpm-lock.yaml`, or `yarn.lock`) to be committed to the root of your repository. It uses the lock file to detect your package manager and install dependencies with a frozen, reproducible install command.
</Note>

## Setup

<Steps>
  <Step title="Install Cypress as a dev dependency">
    If you have not already done so, add Cypress to your project:

    ```bash
    npm install --save-dev cypress
    ```

    Confirm that `cypress` appears in the `devDependencies` section of your `package.json` and that your lock file (`package-lock.json`) is committed to the repository. The action uses the [Cypress Module API](https://on.cypress.io/module-api), which requires Cypress to be a local dependency — not a global install.
  </Step>
  <Step title="Create your workflow file">
    Create `.github/workflows/ci.yml` in your repository with the following content.

    **E2E tests:**

    ```yaml .github/workflows/ci.yml
    name: End-to-end tests
    on: push
    jobs:
      cypress-run:
        runs-on: ubuntu-24.04
        steps:
          - name: Checkout
            uses: actions/checkout@v6
          - name: Cypress run
            uses: cypress-io/github-action@v7
    ```

    This is the minimal working workflow. The action installs your dependencies (with caching) and runs all Cypress E2E tests.

    **Component tests:**

    To run [Cypress Component Testing](https://on.cypress.io/component-testing) instead, add `component: true`:

    ```yaml .github/workflows/ci.yml
    name: Component tests
    on: push
    jobs:
      cypress-run:
        runs-on: ubuntu-24.04
        steps:
          - name: Checkout
            uses: actions/checkout@v6
          - name: Cypress run
            uses: cypress-io/github-action@v7
            with:
              component: true
    ```
  </Step>
  <Step title="Push and watch your tests run">
    Commit your workflow file and push to GitHub:

    ```bash
    git add .github/workflows/ci.yml
    git commit -m "Add Cypress CI workflow"
    git push
    ```

    Navigate to the **Actions** tab of your GitHub repository to see the workflow run. The action installs your dependencies, restores the Cypress binary from cache if available, and executes your tests. Results appear in the job log and in the job summary tab.
  </Step>
</Steps>
